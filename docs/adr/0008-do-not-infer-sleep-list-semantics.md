# Do Not Infer Sleep List Semantics

DeskPilot documentation names future Sleep Lists but does not define how they differ from the implemented per-Category Archive. No time threshold, expiry, automatic movement or reminder schedule has been agreed.

Until that distinction is explicit, DeskPilot must not add a second hidden tab state or time-based behavior under the Sleep List name. Archive already provides non-destructive removal, review, restore and explicitly confirmed permanent deletion; a separate Sleep List requires a concrete additional user outcome before implementation.
