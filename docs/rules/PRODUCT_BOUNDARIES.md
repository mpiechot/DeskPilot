# Product Boundaries

This document defines the boundaries that Codex must not cross.

## Fixed Product Goal

DeskPilot is a local desktop control panel for:

- browser session management
- workflow switching
- reducing browser tab clutter
- reliable session restoration
- future desk automation features

This product goal is fixed.

## Codex May Change

Codex may:

- change the implementation
- change the architecture
- change the project plan
- change milestones
- change issue priorities
- split or merge tasks
- postpone features
- simplify features
- redesign the user interface
- recommend technical changes

## Codex May Not Change

Codex must not:

- change the product vision
- change the target users
- remove browser session management
- turn the project into a generic launcher
- turn the project into a cloud service
- require external accounts
- require subscriptions
- require internet connectivity
- remove local-first operation
- remove user ownership of data

## Local First Principle

DeskPilot must always function locally.

The user owns the data.

Browser sessions must remain accessible without cloud services.

## Human Authority

The user may always override technical decisions.

Codex may argue for a different approach, but the final product vision belongs to the user.
